01 — Setup

clone the repo, drop into the folder, install the deps. that's literally it.

git clone https://github.com/blucrew/7dtd-admin-toolkit.git
cd 7dtd-admin-toolkit
pip install -r requirements.txt

Your .env file

copy .env.example to .env and fill in your server details. this is what auto-fills the connection fields every time you open the tool so you don't have to type it every session.

TDTD_HOST=your.server.ip
TDTD_PORT=8081
TDTD_PASS=yourpassword
never commit .env. it's already in .gitignore. just don't. your password ends up on github forever and that's a bad day.

Profiles

if you're connecting to multiple servers — say your own and someone else's — save each one as a profile via the 💾 button in the top bar. next time you open it, just pick from the dropdown. one click, you're in.

the 7dtd_profiles.json that gets created locally is also gitignored. your passwords stay on your machine.

02 — Connecting to a server

  1. run python 7dtd_manager.py — the .env pre-fills host, port, and password automatically

  2. hit ⚡ Connect — it authenticates, then pulls the full command list from the server's help output and builds the tabs

  3. the dot in the top-right goes green when you're in. the status bar shows the server address.

auto-reconnect is on by default. if the connection drops unexpectedly the tool counts down 10 seconds and tries again. you'll see it in the status bar. if you want to actually disconnect, use the ✖ button — that stops the auto-retry.

keepalive runs every 60 seconds in the background to stop the server from timing you out. you won't notice it.

03 — Running commands

the right panel auto-populates with every command your server supports, sorted into tabs — General, World, Time, Players, Admin, and an All tab if you need to browse everything.

Clicking a command

  1. click any command button — the top panel shows what it does and the syntax

  2. if it needs parameters, a text field appears. fill it in.

  3. hit ⚡ Initiate — confirmation dialog pops up so you don't accidentally shut the server down or boot ziggy mid-fight

Quick bar

the bar under the top row has one-click buttons for the commands you'll spam most: saveworld, shutdown, listplayers, gettime, kick, ban. no navigation, no tabs, just click.

Favourites

right-click any command button and add it to favourites. it shows up in the quick bar next to the defaults with a ⭐ prefix. remove it the same way. saves to favourites.json locally so it persists across sessions.

Raw input

the input bar at the bottom takes any raw command. type it and hit enter. no confirmation dialog — it just sends. useful when you know exactly what you're doing and don't want the click flow.

command history: hit ↑ and ↓ in the raw input bar to cycle through everything you've sent this session. same as a terminal.

04 — Players panel

below the terminal there's a live player list. it polls listplayers every 10 seconds and shows everyone online with their HP, coords, and death count.

Selecting a player

click a row — that player is now selected. when you open a command that needs a player ID (kick, teleport, give, etc.) the ID pre-fills automatically. saves you copying it manually from the terminal.

Right-click quick actions

ActionWhat it does
📍 TP to [name]teleports you to that player

05 — Terminal filter & history

Filter

the search bar above the terminal filters output in real time. type any keyword — player name, command name, whatever — and it shows only matching lines. the full log is still in memory, clear the field to get everything back.

useful when you're trying to track one player or watch for a specific error while the rest of the server chatter scrolls past.

06 — Horde test

the 🧟 Horde tab in the right panel. it spawns waves of zombies from all directions around your base on command. built it to test defences — works great for just causing chaos too.

Launching a horde

  1. set your base X, Y, Z coordinates — this is the centre point everything spawns around

  2. set the radius — how far out from centre zombies spawn (default 35 blocks)

  3. pick a level

  4. hit ⚡ Launch Horde — confirm and it starts

L2 — + Dogs & Birds
L3 — + Ferals & Demos

demolishers. that blinking light on their back is a bomb pack. it explodes when they die. shoot them from a distance. don't rocket them next to your wall. you'll find out why once. that's usually enough.

Progress + countdown

while a horde is running you'll see a live progress bar (Wave 3 / 8 ███░░░░░░░) and a countdown to the next wave. the ⏹ Stop button kills it at any point — sends an abort message to the server chat so people know.

07 — Configuring waves

the ⚙ Configure sub-tab inside the Horde tab is where you build your wave setup. everything here is fully editable and saves to horde_waves.json locally.

Wave list

each wave has a name (rename it whatever you want), a direction dropdown (N, NW, W, SW, S, SE, E, NE, CENTER, ABOVE), and one or more spawn rows.

each spawn row is an entity type and a count. the spinbox next to it is the up/down clicker for the number.

controls per wave:

  • ↑ / ↓ — reorder waves
  • 🗑 — delete the whole wave
  • + Add Spawn — add another entity type to this wave
  • 🗑 on a spawn row — remove just that spawn entry

+ Add Wave at the top creates a new empty wave you can name and configure from scratch.

Break waves

the Break Waves (L2+) section below the wave list is the same format — entity type, count, and now a direction per row. default is dogs spawning at N/S/E/W and vultures above centre. change the types and counts however you want.

the L3 Extra Break section fires 4 seconds after the dog/bird break. screamers, demolishers, and ferals by default.

hit 💾 Save Config when you're happy with your setup. it persists between sessions. ↺ Reset to Defaults puts the original 8-wave setup back if you've gone too far down a rabbit hole.

08 — Presets

you can save named wave configurations and swap between them from the 🚀 Launch tab.

  1. build a wave setup in ⚙ Configure

  2. go back to 🚀 Launch, hit Save As… in the presets row, give it a name

  3. load it any time from the dropdown → Load

  4. 🗑 next to the dropdown deletes the selected preset

good for keeping a few ready: a quick weeknight test, a proper Friday night nightmare run, something you built specifically to grief ziggy.

10 — CLI / standalone horde

horde_test.py runs the horde without the GUI. useful if you want to fire one off from a terminal while you're already in-game and don't want to tab out to the full tool.

python horde_test.py 1   # level 1
python horde_test.py 2   # level 2
python horde_test.py 3   # level 3 — good luck

it reads from the same .env as the GUI. base coords are set at the top of the script — update bx, by, bz to wherever your base is.

this one doesn't ask for confirmation. run it and the first wave goes immediately. don't mistype the level.
